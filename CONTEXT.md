# Binaried domain language

- **Task**: A single unit of planned work owned by the demo user. It has a title, optional description, status, due date, and creation date.
- **Demo User**: The sole v1 identity, `admin`, established by the hardcoded login credentials.
- **Status**: The current task state: `todo`, `in-progress`, or `done`.
- **Due Date**: The required calendar date by which a task should be completed. A past due date represents overdue work.
